# KEY LEARNINGS - NOVEMBER 2025

## Technical Lessons
1. File persistence requires understanding source vs. runtime locations
2. Google Drive sync can conflict with active edits
3. Claude cannot read markdown from Drive - must convert to Google Docs
4. Surgical edits prevent token-consuming rewrites
5. Electron preload.js must expose all required functions

## Protocol Lessons
1. Session startup verification prevents wrong-project work
2. EOS documentation must be executed, not just planned
3. Design standards must be enforced, not just documented
4. Handoff documents enable seamless continuation
5. Token monitoring prevents mid-session failures

## Human Lessons
1. Marathon sessions (13-16 hours) are productive when focused
2. Direct communication works better than corporate politeness
3. "Surgical edits only" respects existing work
4. Break reminders protect health during long sessions
5. Trust must be earned through consistent execution
