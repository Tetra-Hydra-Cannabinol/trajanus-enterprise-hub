## üí° LEARNING STRATEGIES THAT WORK

### 1. The 80/20 Rule
**Focus on the 20% that gets you 80% of results:**

**HTML:** Learn these 15 tags first
```html
<html>, <head>, <body>
<h1>-<h6>, <p>, <a>
<div>, <span>
<ul>, <ol>, <li>
<img>, <form>, <input>
```

**CSS:** Master these concepts first
- Selectors (class, ID, element)
- Box model (margin, padding)
- Flexbox (for layouts)
- Colors and fonts

**JavaScript:** Learn these first
- Variables (let, const)
- Functions
- document.querySelector()
- addEventListener()
- if/else statements

**Python:** Learn these first
- Variables and data types
- Functions
- Lists and dictionaries
- For loops
- File operations

### 2. Code Every Day
**15-30 minutes is better than 3 hours on Sunday:**

- Set a timer for 15 minutes
- Code something small
- Track your streak
- Miss a day? Start again tomorrow

**Your advantage:** You're already coding in these sessions! Just add 15 min/day on your own.

### 3. Build Real Projects
**Don't just do tutorials - build things you'll use:**

‚úÖ **Good practice projects:**
- Automate your file organization
- Create a PM checklist webpage
- Build a project status dashboard
- Write a script to generate reports

‚ùå **Less useful projects:**
- Generic todo list (done to death)
- Calculator (not relevant to your work)
- Random tutorial projects

**Your advantage:** You have REAL problems to solve with the Command Center!

### 4. Type, Don't Copy-Paste
**Muscle memory is real:**

When following tutorials:
- Type every character yourself
- Make intentional typos and fix them
- Experiment with modifications
- Break things and fix them

This feels slower but you learn 10x faster.

### 5. Use Your Own Examples
**Replace tutorial examples with your own:**

Tutorial says:
```python
name = "John"
age = 30
```

You type:
```python
project_name = "SOUTHCOM Guatemala"
completion_percentage = 75
```

**Makes it stick because it's relevant to YOUR world.**

### 6. Teach Someone Else
**Best way to solidify learning:**

- Explain concepts to Tom
- Write comments in your code explaining what you're doing
- Create your own cheat sheets
- Document your learning in Living Documents

**When you can explain it, you know it.**

## üÜò TROUBLESHOOTING COMMON ISSUES

### "I don't understand something"
1. Try a different resource (W3Schools vs MDN vs YouTube)
2. Search Stack Overflow (99% chance someone asked same question)
3. Ask in your next Claude session
4. Break it into smaller pieces
5. Build a simple example to test understanding

### "My code doesn't work"
1. Read error message carefully
2. Check browser console (F12)
3. Add console.log() statements (JavaScript) or print() (Python)
4. Comment out code to isolate problem
5. Search error message on Google
6. Ask Claude for debugging help

### "It works in tutorial but not my code"
1. Compare your code character-by-character
2. Check spelling and capitalization
3. Verify file paths are correct
4. Make sure JavaScript file is loaded
5. Check if function names match exactly

### "I'm stuck and frustrated"
1. Take a break (seriously - walk away for 10 min)
2. Explain problem out loud (rubber duck debugging)
3. Start from scratch with simplest version
4. Ask for help (Claude, Stack Overflow, forums)
5. Remember: Every programmer gets stuck constantly!
