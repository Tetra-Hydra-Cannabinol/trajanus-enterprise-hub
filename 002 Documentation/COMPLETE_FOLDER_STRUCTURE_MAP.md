## DAILY ENTRY STRUCTURE EXAMPLE

Each Living Document type follows this pattern:

```
[Type]_MASTER.md              ← Running compilation of all entries
[Type]_Daily_Entries/         ← Folder containing individual daily files
├── 2025-11-01_[Type].md     ← November 1 entry
├── 2025-11-02_[Type].md     ← November 2 entry
├── 2025-11-03_[Type].md     ← November 3 entry
├── ...                       ← One file per calendar day
└── 2025-12-31_[Type].md     ← December 31 entry
```

**Important:** Every calendar day gets a file, even if "No session on [date]"

## FILE NAMING CONVENTIONS

### Daily Entries
Format: `YYYY-MM-DD_[Document_Type].md`
Examples:
- `2025-12-04_Operational_Journal.md`
- `2025-12-04_Technical_Journal.md`
- `2025-12-04_Personal_Diary.md`

### MASTER Files
Format: `[Document_Type]_MASTER.md`
Examples:
- `Operational_Journal_MASTER.md`
- `Technical_Journal_MASTER.md`
- `Code_Repository_MASTER.md`

### Session Files
Format: `YYYY-MM-DD_Session_Summary.md`
Example: `2025-12-04_Session_Summary.md`

## ACCESS PATTERNS

### For Session Startup
1. Read: `01-Core-Protocols/OPERATIONAL_PROTOCOL.md`
2. Read: `08-EOS-Files/[Latest]_Session_Summary.md`
3. Read: `03-Living-Documents/[Type]_MASTER.md` files as needed

### For Session Closeout
1. Create: `03-Living-Documents/[Type]_Daily_Entries/[Date]_[Type].md` (all 11 types)
2. Update: `03-Living-Documents/[Type]_MASTER.md` (append new content from daily entries)
3. Create: `08-EOS-Files/[Date]_Session_Summary.md`

### For Historical Research
1. Search: `03-Living-Documents/[Type]_MASTER.md` for quick overview
2. Browse: `03-Living-Documents/[Type]_Daily_Entries/` for specific dates
3. Reference: `07-Session-Journal/` for complete session contexts
